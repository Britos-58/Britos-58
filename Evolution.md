## 📅 2022 | The Foundation — First Web Profile

**Project Goal**  
Create a personal website where I could present everything I had done, what I was currently doing, and what I planned to do in the future, including projects and communities I was connected to.

**Sub-Goal**  
Learn programming by building a functional website.

### 🔍 Project Origin
At the time I was searching for free platforms to learn **Python**, because my main goal was to create a game. During that search I discovered **freeCodeCamp**.

While studying Python, I noticed the web development learning paths. Since web knowledge would also be useful for future projects, I decided to explore the **Responsive Web Design Certification**.

The material felt surprisingly intuitive and enjoyable, so I decided to attempt building a **personal portfolio website**.  
However, the project was never fully completed or deployed due to school responsibilities and other personal commitments.

###  HTML & CSS
I structured the project using **separate files for each page**.

Each page contained:
- its own `.html` file  
- its own `.css` stylesheet

This approach helped me stay organized because, at the time, large code files were difficult for me to read and navigate. Splitting the code into smaller sections made the learning process easier.

Each completed page created a strong sense of progress and motivation.

###  JavaScript
JavaScript files were also separated for each page.

Compared to HTML and CSS, JavaScript was significantly harder to understand. Even with AI assistance available at the time, translating ideas into working logic was challenging.  

This difficulty eventually reduced my motivation to continue the project.

###  VSCode
The main development environment was **VSCode**, where I used a preview extension to view the website locally.

During development I often built components in **CodePen** first, because it allowed faster experimentation. Once the components worked properly, I transferred them to the main project in VSCode.

###  Learning Outcome
Although the project was not completed, I managed to create visually clean interfaces with limited animations.

For a beginner project at that stage, the website design was relatively elegant compared to many basic beginner sites.  

The project was ultimately abandoned due to academic workload, personal studies, and other parallel activities.

## 📅 2024 | Systems & Environments — Private Web Host

**Project Goal**  
Recreate my website with more content and host it online.

**Sub-Goal**  
Understand the basic structure of a functional web environment.

### 🔍 Project Origin
At the beginning of the year I created a community and wanted to add a more professional element to it by building a website connected to the community.

The website would contain:
- community information  
- my portfolio  
- other relevant resources

Because I had already experimented with HTML and CSS in the past, I decided to attempt the project again using a more modern development structure.

###  React
React introduced a more complex development structure compared to the HTML and CSS approach I was previously familiar with.

However, it allowed me to experiment with the type of visual interactions and dynamic interfaces I had always wanted to build without significantly affecting performance.

###  Node.js
Through Node.js I began experimenting with backend environments and understanding how JavaScript can be used outside the browser.

This helped me start exploring how web applications manage logic and server-side processes.

###  PostgreSQL
For the first time I experimented with a **database-driven structure**.

This part of the project was extremely challenging. I spent a lot of time researching tutorials, documentation, and occasionally using AI assistance to understand database connections and structure.

The database was initially hosted locally and later experimented with through a free online database service.

###  Environment Management
The host environment remained private during development.  
The original idea was to complete the project first and then publish the final website link inside the community platform.

###  Learning Outcome
This project showed me that there are multiple ways to build the same type of system.

It also revealed the deeper complexity behind modern technologies. During this process I improved my ability to design more dynamic interfaces with visual effects across multiple components.

The project was not publicly launched because the free database services had limitations, and creating a stable structure without investing financially was difficult at that stage.

Despite that, the project served as an important learning experience for understanding web architecture and development workflows.

## 📅 2025 | Performance & Native Applications — Application Build

**Project Goal**  
Build the most complex application I can develop and aim to release it before the end of **2026**.

**Sub-Goal**  
Learn advanced development models, system structures, and architectural strategies.

### 🔍 Project Origin
Now that I am entering the professional stage of my life and have clearer project goals, I decided to start developing a larger application concept.

The long-term objective of this application is to address certain structural problems in society. The project is designed to evolve continuously over time.

Even though the project may continue for many years, it serves as a personal development environment where I can practice and improve skills in areas such as:
- programming  
- DevOps concepts  
- cybersecurity fundamentals  
- system architecture

The idea is to combine **deep internal complexity** with a **simple and intuitive user experience (UI/UX)**.

###  React
Used to explore strategies for building structured and visually engaging interfaces.

###  Vite
Used as the development environment to improve development speed and project organization.

###  Database
The project initially started using **SQLite**.

It is currently being migrated toward **PostgreSQL combined with Tailscale networking**, allowing secure connectivity between devices without relying on the same Wi-Fi network.

###  System Optimization
This project also focuses on understanding how software interacts with hardware resources and system performance.

###  Git
Parts of the project will gradually be published through Git repositories to document development progress.

###  Learning Outcome
This phase represents the beginning of a long-term technical learning path.  
Future knowledge, experimentation, and improvements will continue to be integrated into this project.

This progression reflects my growing interest not only in building software, but also in understanding the underlying systems that make modern technology function.
